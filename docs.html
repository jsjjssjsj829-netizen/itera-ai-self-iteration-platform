公开文档链接

这就是可以发给客户的公开接入文档地址。生产环境上线后，域名会自动变成你的公网域名。

http://127.0.0.1:8787/docs

3 步完成接入

1 填写产品信息

在控制台填写产品名称和网站网址。平台会自动规范化网址并生成允许来源。

2 生成 API Key

提交后系统会为该网站生成独立 API Key，并绑定到当前项目。

3 复制嵌入代码

把一行脚本粘贴到客户网站的 HTML 中，建议放在 `` 前或全站模板中。

嵌入代码

控制台会自动把客户自己的 API Key 填进去。下面是格式示例：

HTML 
<script src="http://127.0.0.1:8787/widget.js" data-key="sdk-your-project-key" defer></script>

接入后，SDK 会自动发送一次 `sdk_loaded` 心跳，用来确认脚本已成功加载。

验证接入是否成功

1 刷新客户网站

打开已粘贴脚本的网站页面，让 SDK 自动上报首次心跳。

2 回到控制台

项目接入面板里会显示最近成功来源、通过信号数和拦截信号数。

3 发送测试信号

点击控制台里的“发送测试信号”，确认 API Key、域名来源和 Signals API 都正常。

Signals API

如果客户不想使用页面浮窗，也可以由自己的前端、后端或客服系统直接调用 Signals API。

POST /api/signals 
curl -X POST http://127.0.0.1:8787/api/signals \
  -H "Content-Type: application/json" \
  -H "X-Itera-SDK-Key: sdk-your-project-key" \
  -d '{"type":"feedback","source":"用户反馈","page":"https://customer.example.com/checkout","data":{"text":"支付按钮点击后没有反应"}}'
type信号类型，例如 `feedback`、`client_error`、`performance`、`api_failure`。
source信号来源，例如“用户反馈”“客服工单”“前端错误”“AI 巡检”。
page信号发生页面，用于校验来源和定位问题。
data.text用户原始反馈或系统错误摘要，平台会自动分类并生成迭代任务。

输出 Webhook

客户如果希望把 PR、发布计划、灰度推进和回滚结果同步回自己的系统，可以在控制台配置输出 Webhook。

  • 平台会发送 `webhook.test`、`pr.opened`、`release.planned`、`release.promoted`、`release.rolled_back` 等事件。
  • 每次请求都会带 `X-Itera-Event`、`X-Itera-Delivery`、`X-Itera-Signature-256`。
  • 签名使用项目 API Key 对请求体做 HMAC-SHA256，客户后端可以用它验证来源。

常见问题

看不到信号确认脚本已粘贴到客户网站、API Key 正确、客户网站域名和项目网址一致。
403 拒绝通常是 SDK Key 错误或浏览器来源不在允许域名内。重新生成接入代码后再粘贴。
429 限流单项目默认每分钟最多接收 120 条信号，可通过 `SIGNAL_RATE_LIMIT_PER_MINUTE` 调整。
公网访问本地 `127.0.0.1` 只能自己电脑访问。正式给客户使用时需要部署到公网 HTTPS 域名。
